# Runix Tech Contact Flow

---

# 1. Purpose

The contact system should help Runix Tech collect serious project inquiries in a clean, structured way while keeping the experience simple for the visitor.

The form should not feel intimidating, but it should gather enough information to understand the project properly.

---

# 2. Main Contact Goal

Collect project inquiries for:
- websites
- landing pages
- dashboards
- web apps / MVPs
- redesigns
- UI / frontend-focused work

---

# 3. Contact Methods

## Primary Method
Project inquiry form on the Contact page.

## Secondary Methods
- email
- WhatsApp
- optional social links if relevant

---

# 4. Recommended Form Fields

## Required
- Name
- Email
- Project Type
- Project Details

## Recommended Optional / Semi-Required
- Company / Brand Name
- Budget Range
- Timeline
- Existing Website (if any)

---

# 5. Suggested Project Type Options

- Business Website
- Portfolio Website
- Landing Page
- Dashboard / Admin Panel
- Web App / MVP
- Website Redesign
- Not Sure Yet

---

# 6. Suggested Budget Range Options

- Under ₹15,000
- ₹15,000 – ₹30,000
- ₹30,000 – ₹60,000
- ₹60,000+
- Let’s discuss

---

# 7. Suggested Timeline Options

- As soon as possible
- Within 2 weeks
- Within 1 month
- Flexible / exploring options

---

# 8. Form UX Rules

- labels must be clear
- spacing should be clean and readable
- fields should not feel cramped
- the form should visually match the premium dark design system
- required fields should be obvious
- error messages should be clean and non-aggressive
- mobile usability must be good

---

# 9. Submission Behavior

When the user submits:
1. validate fields
2. send inquiry to configured email / backend
3. show success state
4. optionally reset form or keep data depending on UX decision
5. log inquiry if a backend system exists later

---

# 10. Success State Copy

Example:
“Thanks for reaching out. Your project inquiry has been received. Runix Tech will review the details and get back as soon as possible.”

Optional addition:
“If your project is time-sensitive, mention that in the details or use direct contact as well.”

---

# 11. Direct Contact Block

The Contact page should also include a small direct contact section with:
- email
- WhatsApp (optional)
- short response expectation note

Example note:
“For serious project inquiries, include as much context as possible so the response can be more useful.”

---

# 12. Spam / Quality Control Notes

Potential future anti-spam options:
- honeypot field
- rate limiting if backend exists
- lightweight captcha if needed later

Do not overcomplicate V1 unless spam becomes a real issue.

---

# 13. Contact UX Goal

The visitor should leave the Contact page feeling:
- it was easy to reach out
- Runix Tech looks professional
- their inquiry was taken seriously
