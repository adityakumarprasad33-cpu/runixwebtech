# Runix Web Technologies  Prompt System

This file stores the prompt framework for building and refining the Runix Web Technologies  website with AI coding tools such as Cursor, Claude, Gemini, Lovable, Bolt, v0, or similar systems.

# 1. Master Build Prompt

## MASTER PROMPT

Build a premium dark-mode website for **Runix Web Technologies **, a web development and digital product brand that creates modern websites, dashboards, landing pages, web apps, and redesigns.

The website should feel like a polished, high-end software/product studio website rather than a generic freelancer portfolio. It should be visually inspired by premium SaaS/product websites, with a clean dark interface, strong typography, rounded surfaces, subtle borders, refined motion, and a modern showcase-first layout.

## Core goals
- Present Runix Web Technologies  as a premium web development / product engineering brand
- Showcase projects and live websites in a polished way
- Explain services clearly
- Convert visitors into project inquiries
- Feel modern, intentional, responsive, and production-quality

## Required pages
- Home
- Work
- Services
- About
- Contact

## Required homepage sections
- premium hero with strong positioning
- CTA buttons
- featured project showcase
- services overview
- why choose Runix Web Technologies 
- process section
- about snapshot
- final contact CTA

## Required Work page features
- project grid / showcase cards
- category / tag support
- project summary and actions
- ability to open live preview or case study links

## Required Services page features
- clear service cards and detailed service explanations
- premium layout
- CTA to start a project

## Required Contact page features
- premium contact form
- direct contact block
- success state

## Design direction
- dark premium interface
- modern typography
- large rounded cards and sections
- subtle glass / layered panel feel where useful
- restrained accent color
- premium hover states
- elegant animations, not flashy ones

## Important build rules
- use reusable components
- use clean TypeScript + Tailwind structure
- keep content modular and data-driven where possible
- fully responsive
- avoid generic template feeling
- no fake testimonials or fake logos
- no excessive gradients or noisy effects

## Showcase requirement
Build a project showcase system with cards and support for a browser-style preview modal or a clean “open live project” flow. Do not load many heavy live embeds on page load. Prefer thumbnails and lazy-loaded preview behavior.
